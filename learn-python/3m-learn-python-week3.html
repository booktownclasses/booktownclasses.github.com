Day 1 Lists: Creation, Indexing & Slicing

Learning Objectives: Understand what lists are. Create lists with different data types. Access items using positive and negative indexing. Slice lists to extract sublists. Use len() and in operator.

Creating Lists

day1_create.py
# Empty list
empty = []

# List of integers
numbers = [10, 20, 30, 40]

# Mixed data types
mixed = [1, "hello", 3.14, True]

# Nested list
matrix = [[1, 2], [3, 4]]

print(len(numbers))     # 4
print(10 in numbers)    # True

Indexing (positive & negative)

day1_indexing.py
fruits = ["apple", "banana", "cherry", "date"]
print(fruits[0])      # apple  (first)
print(fruits[2])      # cherry (third)
print(fruits[-1])     # date   (last)
print(fruits[-2])     # cherry (second last)

Slicing Lists

day1_slicing.py
nums = [0, 1, 2, 3, 4, 5]
print(nums[1:4])    # [1, 2, 3]   index 1 to 3
print(nums[:3])     # [0, 1, 2]   start to index 2
print(nums[2:])     # [2, 3, 4, 5] index 2 to end
print(nums[::2])    # [0, 2, 4]   every second element
print(nums[::-1])   # [5, 4, 3, 2, 1, 0]  reverse

🧑‍💻 Exercise: Slice & Dice

Given the list colors = ["red", "green", "blue", "yellow", "purple"], write code to: 1) Print the first three colors. 2) Print the last two colors. 3) Print the list in reverse order.

day1_exercise_solution.py
colors = ["red", "green", "blue", "yellow", "purple"]
print(colors[:3])       # ['red', 'green', 'blue']
print(colors[-2:])      # ['yellow', 'purple']
print(colors[::-1])     # ['purple', 'yellow', 'blue', 'green', 'red']
💡 Instructor Tip: Visualize a list as numbered boxes. The first box is index 0. Negative indices count from the right. Slicing always returns a new list; it does not modify the original.

Day 2 List Methods & Operations

Learning Objectives: Add, remove, and modify elements using list methods. Combine lists with + and extend(). Sort, reverse, and copy lists. Avoid common mutation pitfalls.

Adding & Removing Elements

day2_add_remove.py
fruits = ["apple", "banana"]
fruits.append("cherry")       # adds at end -> ['apple','banana','cherry']
fruits.insert(1, "mango")     # insert at index 1 -> ['apple','mango','banana','cherry']
fruits.remove("banana")       # removes first occurrence -> ['apple','mango','cherry']
popped = fruits.pop()         # removes last, returns it -> 'cherry'; list: ['apple','mango']
fruits.pop(0)                 # removes index 0 -> 'apple'; list: ['mango']
del fruits[0]                 # deletes index 0 without returning
print(fruits)                 # []

Sorting, Reversing & Copying

day2_sort_copy.py
nums = [3, 1, 4, 1, 5, 9]
nums.sort()              # in-place sort -> [1,1,3,4,5,9]
nums.sort(reverse=True)  # descending -> [9,5,4,3,1,1]
nums.reverse()           # reverse current order -> [1,1,3,4,5,9]

# Creating a copy (not a reference!)
original = [1, 2, 3]
copy1 = original.copy()    # method 1
copy2 = original[:]        # method 2 (slicing)
copy1.append(4)
print(original)   # [1,2,3] (unchanged because it's a true copy)

Combining Lists

day2_combine.py
a = [1, 2]
b = [3, 4]
c = a + b            # [1,2,3,4] (creates new list)
a.extend(b)          # a becomes [1,2,3,4] (mutates a)
print(a)

🧑‍💻 Exercise: Shopping List Manager

Create an empty list cart. Ask the user to add 3 items one by one using append(). Then remove the second item using pop(). Finally sort the list and print it.

day2_exercise_solution.py
cart = []
cart.append(input("Item 1: "))
cart.append(input("Item 2: "))
cart.append(input("Item 3: "))
print("Cart before:", cart)
cart.pop(1)          # remove index 1 (second item)

Item 1: milk Item 2: eggs Item 3: bread Cart before: ['milk', 'eggs', 'bread'] Sorted cart: ['bread', 'milk']
⚠️ Beware of references: Assignment b = a does NOT copy a list; both variables point to the same object. Use .copy() or [:] for a shallow copy.

Day 3 Tuples: Immutable Sequences

Learning Objectives: Understand tuple immutability. Create tuples with and without parentheses. Access tuple items (indexing & slicing). Use tuple methods (count(), index()). Unpack tuples elegantly.

Tuple Basics

day3_tuple_basics.py
# Creating tuples
empty = ()
single = (5,)          # comma is required for single element!
point = (3, 7)
mixed = (1, "hello", 3.14)

# Without parentheses (tuple packing)
coords = 10, 20        # (10, 20)
print(type(coords))    # 

# Indexing works same as lists
print(point[0])        # 3
print(point[-1])       # 7

Immutability — Tuples Cannot Be Changed

day3_immutable.py
t = (1, 2, 3)
# t[0] = 10            # ❌ TypeError: 'tuple' object does not support item assignment
# t.append(4)          # ❌ AttributeError

# But you can create a new tuple by concatenation
t = t + (4, 5)          # (1,2,3,4,5) – new tuple assigned to t
print(t)

Tuple Methods & Unpacking

day3_unpacking.py
nums = (2, 4, 2, 6, 2)
print(nums.count(2))    # 3 (occurrences of 2)
print(nums.index(6))    # 3 (first index of 6)

# Tuple unpacking
point = (10, 20)
x, y = point
print(x, y)             # 10 20

# Swapping variables elegantly (uses tuple packing/unpacking)
a, b = 5, 10
a, b = b, a
print(a, b)             # 10 5

🧑‍💻 Exercise: Student Record

Create a tuple student with (name, age, grade). Unpack it into three variables and print them. Then use count() to see how many students have grade 'A' in a list of tuples.

day3_exercise_solution.py
# Single student unpacking
student = ("Riya", 14, "A")
name, age, grade = student
print(f"{name}, age {age}, grade {grade}")

# Counting 'A' grades in a list of tuples
students = [("Riya", "A"), ("Amit", "B"), ("Sara", "A"), ("Kunal", "C")]
a_count = sum(1 for _, g in students if g == "A")
print(f"Students with A grade: {a_count}")
💡 Why tuples? Use tuples for data that shouldn't change (e.g., coordinates, days of the week). They are faster than lists and can be used as dictionary keys (lists cannot).

Day 4 List Comprehension

Learning Objectives: Write concise list transformations using comprehensions. Add conditional filtering (if) and if-else inside comprehensions. Handle nested loops in comprehensions. Compare with traditional loops.

Basic Syntax

day4_basic_comp.py
# Without comprehension
squares = []
for i in range(1, 6):
    squares.append(i ** 2)
print(squares)   # [1, 4, 9, 16, 25]

# With list comprehension
squares_comp = [i ** 2 for i in range(1, 6)]
print(squares_comp)   # [1, 4, 9, 16, 25]

# From a list
words = ["hello", "world", "python"]
lengths = [len(w) for w in words]   # [5, 5, 6]

With Conditionals

day4_conditionals.py
# Filtering: only even numbers
evens = [x for x in range(1, 11) if x % 2 == 0]
print(evens)   # [2, 4, 6, 8, 10]

# if-else inside expression (note different position)
labels = ["even" if x % 2 == 0 else "odd" for x in range(1, 6)]
print(labels)  # ['odd', 'even', 'odd', 'even', 'odd']

Nested Loops in Comprehension

day4_nested_comp.py
# Generate coordinate pairs
pairs = [(x, y) for x in range(1, 3) for y in range(1, 3)]
print(pairs)   # [(1,1), (1,2), (2,1), (2,2)]

# Multiplication table (list of lists)
table = [[i*j for j in range(1, 4)] for i in range(1, 4)]
print(table)   # [[1,2,3], [2,4,6], [3,6,9]]

🧑‍💻 Exercise: Filter and Transform

Given a list of numbers nums = [5, 12, 17, 24, 35], create a new list containing the string "high" if the number is > 15 else "low". Use a list comprehension with an if-else expression.

day4_exercise_solution.py
nums = [5, 12, 17, 24, 35]
result = ["high" if n > 15 else "low" for n in nums]
print(result)   # ['low', 'low', 'high', 'high', 'high']
⚠️ Readability Note: List comprehensions are powerful but can become unreadable with too many expressions or nested loops. If it's longer than ~70 characters or has complex logic, use a regular for loop instead.

Day 5 Mini-Project: Student Gradebook

Learning Objectives: Build a menu-driven program using lists and tuples. Store student records as tuples inside a list. Perform operations: add student, display all, show average grade, find top student. Apply all Week 3 concepts.

Project Requirements

  1. Store each student as a tuple: (name, marks).
  2. Provide a menu: 1. Add student, 2. View all, 3. Average marks, 4. Top student, 5. Exit.
  3. Use a while loop to keep running until exit.
  4. For "Top student", find the tuple with the highest marks.
  5. Bonus: Use list comprehension where possible (e.g., extracting all marks).
gradebook.py
students = []   # list of (name, marks) tuples

while True:
    print("\n--- Student Gradebook ---")
    print("1. Add student")
    print("2. View all students")
    print("3. Average marks")
    print("4. Top student")
    print("5. Exit")
    choice = input("Enter choice: ")

    if choice == '1':
        name = input("Student name: ")
        marks = float(input("Marks: "))
        students.append((name, marks))
        print("✅ Added.")
    elif choice == '2':
        if not students:
            print("No students yet.")
        else:
            for name, marks in students:
                print(f"{name}: {marks}")
    elif choice == '3':
        if not students:
            print("No students.")
        else:
            all_marks = [m for _, m in students]   # list comprehension
            avg = sum(all_marks) / len(all_marks)
            print(f"Average marks: {avg:.2f}")
    elif choice == '4':
        if not students:
            print("No students.")
        else:
            top = max(students, key=lambda s: s[1])   # s[1] is marks
            print(f"Top student: {top[0]} with {top[1]} marks")
    elif choice == '5':
        print("Goodbye!")
        break
    else:
        print("Invalid choice.")
--- Student Gradebook --- 1. Add student 2. View all students 3. Average marks 4. Top student 5. Exit Enter choice: 1 Student name: Ravi Marks: 87 ✅ Added. ... Average marks: 84.50
🚀 Extension: Add "Search by name" using a loop. Add "Remove student". Store data in a file (Week 4 covers files). Convert the program to use a dictionary for faster lookups (Week 5).

Day 6 GitHub: Push, Pull & Clone

Learning Objectives: Understand remote repositories. Connect a local Git repo to GitHub. Push code to GitHub. Pull changes from GitHub. Clone an existing GitHub repository.

Why GitHub?

GitHub hosts your Git repositories online, enabling backup, sharing, and collaboration. Developers can work on the same project from anywhere.

Essential GitHub Commands

CommandPurpose
git remote add origin <url>Link your local repo to a GitHub remote
git push -u origin mainUpload local commits to GitHub
git pull origin mainDownload and merge changes from GitHub
git clone <url>Copy a remote repository to your computer
git remote -vShow remote URLs

Pushing an Existing Project

Scenario: You have a local repo (like the gradebook) and want to back it up to GitHub.

terminal — first push
# 1. Create an empty repo on GitHub (without README/.gitignore)
# 2. In your local project folder:
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/yourusername/gradebook.git
git push -u origin main      # -u sets upstream, so next time just 'git push'

Cloning a Repository

terminal — clone
# Get the URL from GitHub (green 'Code' button)
git clone https://github.com/username/repo.git
cd repo
# Now you have the full project, with Git history ready.

Collaboration Flow: Push / Pull

typical daily workflow
# Start of day: pull latest changes from GitHub
git pull origin main

# Work on your code...
# After making changes and committing:
git add .
git commit -m "Added search feature"
git push origin main        # sends your commits to GitHub

# If partner also pushed, your push may be rejected. Then:
git pull origin main        # merge partner's changes first
# resolve any conflicts, then push again.

💡 Pro Tips: Always git pull before starting work. Use git status frequently. For team projects, create branches for features (Week 2 skill) and push those branches to GitHub. Use .gitignore to avoid committing temporary files.
